# Моделирование речной навигации (ЛР №4–6)

Программный комплекс для анализа связности, оптимизации маршрутов и оценки устойчивости речных транспортных сетей. Проект объединяет результаты лабораторных работ №4, №5 и №6 по дисциплине «Алгоритмизация и программирование».

## Спецификация
* **Язык:** C# 13.0
* **Платформа:** .NET 9.0 (Windows)
* **Интерфейс:** Windows Forms
* **IDE:** Visual Studio 2026
* **Тестовое покрытие:** MSTest (проверка ядра графовых алгоритмов и граничных случаев)

##  Описание проекта
Приложение моделирует единую речную логистическую систему, где:
* **Вершины** — города-порты.
* **Рёбра** — судоходные участки рек.
* **Вес ребра** — протяжённость участка в километрах.

В ходе выполнения трёх лабораторных работ функционал был последовательно расширен: от базового обхода графа до поиска оптимальных сетей и выявления критически важных узлов инфраструктуры.

## Функционал по лабораторным работам

### ЛР №4: Базовый анализ связности и обход
* **Поиск в ширину (BFS):** проверка достижимости порта и поиск пути с минимальным числом переходов.
* **Поиск в глубину (DFS):** полный обход доступной речной сети.
* **Компоненты связности:** автоматическое выявление изолированных речных бассейнов.
* **Восстановление маршрута:** построение цепочки городов между двумя портами с подсчётом пройденного расстояния.

### ЛР №5: Оптимизация маршрутов и сетей
* **Алгоритм Дейкстры:** поиск кратчайшего водного пути по суммарному расстоянию (в км).
* **Алгоритм Прима:** построение минимального остовного дерева (МОД) — оптимальной сети судоходных путей, соединяющей все порты с минимальными суммарными затратами на обслуживание.

### ЛР №6: Анализ устойчивости инфраструктуры
* **Точки сочленения (Articulation Points):** выявление критических портов, удаление которых приводит к разрыву единой речной сети на изолированные фрагменты. Позволяет планировать дублирующие маршруты и оценивать риски логистических сбоев.

## Формат входных данных
Программа загружает структуру графа из текстового файла (по умолчанию `river_graph.txt`). Каждая строка описывает один участок реки:
```text
Город_Отправления;Город_Прибытия;Расстояние_в_км
```
*Пример:*
```text
Тверь;Углич;120
Углич;Мышкин;40
Рыбинск;Углич;65
```
*(Граф считается неориентированным, путь симметричен в обе стороны)*

## Тестирование
Логика ядра (`Graph.cs`) покрыта комплексным набором Unit-тестов, проверяющим:
* Корректность парсинга и построения списка смежности.
* Работу BFS/DFS на связных и несвязных графах.
* Точность расчёта кратчайших путей (Дейкстра) и веса МОД (Прим).
* Корректное выявление точек сочленения в графах различной топологии.
* Обработку исключений (несуществующие порты, пустые файлы, циклические зависимости).

## Запуск и клонирование проекта

### 1. Клонирование репозитория
Откройте терминал (Git Bash, PowerShell или CMD) и выполните:
```bash
git clone https://github.com/kkornilov55-glitch/LibraryAPP.git
cd rivers-graph-labs
```

### 2. Настройка среды
* Убедитесь, что установлена **.NET 9.0 SDK**.
* Откройте решение (`*.sln`) в **Visual Studio 2026**.
* Восстановите NuGet-зависимости (автоматически или через `dotnet restore`).

### 3. Подготовка данных
Поместите файл `river_graph.txt` в рабочую директорию проекта (обычно `bin/Debug/net9.0-windows/`) или укажите полный путь к файлу через интерфейс приложения.

### 4. Запуск
* Нажмите `F5` в Visual Studio для запуска в режиме отладки.
* Или выполните из корня проекта:
  ```bash
  dotnet build
  dotnet run
  ```
