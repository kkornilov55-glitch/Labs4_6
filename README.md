# Моделирование речной навигации (ЛР №4)

Программное решение для анализа связности речных путей между городами. Проект выполнен в рамках лабораторной работы №4 по дисциплине «Алгоритмизация и программирование».

## 💻 Спецификация
* **Язык программирования:** C# (версия 12.0)
* **Платформа:** .NET 8.0 (Windows)
* **Тип приложения:** Desktop (Windows Forms / WPF)
* **Среда разработки:** Visual Studio 2022

## 📝 Описание проекта
Приложение моделирует схему речной системы, где:
* **Вершины** — это города-порты.
* **Рёбра** — водные пути (судоходные реки) между ними.
* **Вес ребра** — расстояние в километрах.

Программа позволяет загружать структуру графа из текстового файла и проводить анализ транспортной доступности.

## Основной функционал
В проекте реализованы базовые алгоритмы теории графов:
1.  **Поиск в ширину (BFS):** для определения кратчайшего пути по количеству переходов и проверки достижимости города.
2.  **Поиск в глубину (DFS):** для полного обхода всех доступных водных путей в системе.
3.  **Анализ связности:** автоматическое определение изолированных речных бассейнов (компонент связности).
4.  **Восстановление маршрута:** вывод цепочки городов от начальной до конечной точки.

## Формат входных данных
Программа считывает данные из файла `river_graph.txt`. Каждая строка представляет собой описание одного водного пути в формате:
```text
Город_А;Город_Б;Расстояние
```
*Пример:*
```text
Тверь;Углич;120
Углич;Мышкин;40
```

## Тестирование
Логика обработки графа покрыта **11 Unit-тестами** (MS Test), проверяющими:
* Корректность парсинга файла.
* Работу алгоритмов на связанных и изолированных вершинах.
* Обработку исключений при вводе несуществующих портов.
